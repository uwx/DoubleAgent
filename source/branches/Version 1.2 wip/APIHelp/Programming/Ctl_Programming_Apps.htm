Double Agent API

Using the Control in Applications

You can access the Double Agent server from any programming environment that supports the COM standard or the .NET standard. This includes Visual Basic 6, Visual Basic .NET, Visual C#, Visual C++, JScript, VBScript, and many other programming languages.

For programming languages that support native Windows programming, use Double Agent's native ActiveX® control.

  • To include the control in a form, make sure the language fully supports the ActiveX® Control Interface, and follow its conventions for adding and accessing ActiveX® controls.
  • If the language supports it, you can also use the New operator with the "DoubleAgent.Control" or "DoubleAgent.Control.1.1" ProgID, or the "{1147E530-A208-11DE-ABF2-002421116FB2}" ClassID.

The programming environment may require a declared reference to the control's COM type library:

  • The native control is registered as an ActiveX® Control object. If the programming environment provides a list of registered ActiveX® Control or COM Automation objects, it will appear in the list.
  • A reference copy of the control's COM type library is installed with the Double Agent SDK in folder C:\Program Files\Double Agent\Dev.
  • For programming environments that prefer to retrieve type information from the actual run-time, a copy of the control's COM type library is also included in the control's run-time (DaControl.dll) as an embedded resource.

For programming languages that support the .NET Common Language Runtime (CLR), there are two options:

  1. You can use Double Agent's Control Primary Interop Assembly.
    • It provides the Double Agent Control as simple a .NET object (a subclass of System.Object) that you can instantiate using the New operator (no form required).
    • Use the Control class in the DoubleAgent.Control namespace.
  2. If the language supports the System.Windows.Forms namespace, you can use Double Agent's ActiveX® Control Primary Interop Assembly.
    • It provides the Double Agent Control as a Windows Forms Control (a subclass of System.Windows.Forms.Control) that you can drop in any form.
    • Use the AxControl class in the DoubleAgent.AxControl namespace.

The programming environment may require a declared reference to these assemblies:

  • For a programming environment that retrieves references from the Global Assembly Cache, this assembly is added to the GAC when Double Agent is installed.

    There are different versions of the Primary Interop Assembly for .NET 3.5 or earlier and .NET 4.0 or later.  If you move from .NET 3.5 to .NET 4.0 you should reinstall the Double Agent SDK to get the correct Primary Interop Assembly version installed in the the GAC.

  • If an assembly file is required, a reference copy of these assemblies is installed with the Double Agent SDK in folder C:\Program Files\Double Agent\Dev\v3.5 (for .NET 3.5 or earlier) or C:\Program Files\Double Agent\Dev\v4.0 (for .NET 4.0 or later).
  • You can use a reference to a specific assembly's full path, or add the SDK install location to the programming environment's reference search path.
