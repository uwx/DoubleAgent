Double Agent SDK

Moving from Microsoft® Agent to Double Agent

Applications developed for Microsoft® Agent will continue to work with Double Agent. However, if you want to develop your application for Double Agent instead of Microsoft® Agent, make these simple changes:

  • If you're using the native Agent Control:
    • Replace the Microsoft® Agent Control ProgId Agent.Control or Agent.Control.2 with the Double Agent Control ProgId DoubleAgent.Control or DoubleAgent.Control.1.1
    • Replace the Microsoft® Agent Control ClassId {D45FD31B-5C6E-11D1-9EC1-00C04FD7081F} with the Double Agent Control ClassId {1147E530-A208-11DE-ABF2-002421116FB2}
    • Replace the Microsoft® Agent Control TypeLib AgentObjects {F5BE8BC2-7DE6-11D0-91FE-00C04FD701A5} with the Double Agent Control TypeLib DoubleAgentCtl {1147E550-A208-11DE-ABF2-002421116FB2}
  • If you're using the native Agent Server:
    • Replace the Microsoft® Agent Server ProgId Agent.Server or Agent.Server.2 with the Double Agent Server ProgId DoubleAgent.Server or DoubleAgent.Server.1.1
    • Replace the Microsoft® Agent Server ClassId {D45FD2FC-5C6E-11D1-9EC1-00C04FD7081F} with the Double Agent Server ClassId {1147E500-A208-11DE-ABF2-002421116FB2}
    • Replace the Microsoft® Agent Server TypeLib AgentServerObjects {A7B93C73-7B81-11D0-AC5F-00C04FD97575} with the Double Agent Server TypeLib DoubleAgentSvr {1147E501-A208-11DE-ABF2-002421116FB2}
  • For C and C++ development, replace the the Microsoft® Agent interfaces in AgtSvr.h and AgtCtl.h with the coresponding Double Agent interfaces in DaServerOdl.h and DaControlOdl.h.
    Optionally, you can also use the interface smart pointers in DaControlInterface.h and DaServerInterface.h.

  • For Visual Basic 6 development, replace AgentCtl.dll references with DaControl.dll.
    You can easily change these references in your project (.vbp) and form (.frm) files with a text editor, as follows:

    change: Object = "{F5BE8BC2-7DE6-11D0-91FE-00C04FD701A5}#2.0#0"; "AgentCtl.dll"
    to: Object = "{1147E550-A208-11DE-ABF2-002421116FB2}#1.0#0"; "DaControl.dll"

  • For .NET languages, change the COM Interop Assembly references as follows:

    change: AgentObjects to: DoubleAgent.Control
    change: AxAgentObjects to: DoubleAgent.AxControl
    change: AgentServerObjects to: DoubleAgent.Server
