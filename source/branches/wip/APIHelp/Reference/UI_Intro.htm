The Double Agent User Interface

The Double Agent user interface generally reproduces the Microsoft® Agent user interface. Double Agent version 1.0 reproduces the Microsoft® Agent 2.0 user interface quite faithfully, except that a few components (such as the Listening tip) have been brought up-to-date.

The Double Agent version 1.1 user interface has evolved a bit, but is still similar to Microsoft® Agent. Because Double Agent supports the Microsoft® Agent international language components, the evolution of its user interface is limited.

The Character Window

Normally, Double Agent displays animated characters in their own windows that float over other application windows. A user can move a character's window by dragging the character with the left mouse button. The character image moves with the pointer. An application can also move a character using the MoveTo method.

The Double Agent Control can also be used in contained mode. It appears as a child window within an application and it's character is limited to its window. In this case, the user cannot move the character and the MoveTo method is disabled.

The Pop-up Menu

When the user right-clicks a character, a pop-up menu appears:

* Commands listed are based on the input-active client. For more information on defining commands that appear in the pop-up menu, see the Double Agent API Overview.

** Entries listed are all other applications currently hosting the character. For more information on defining this entry, see the Double Agent API Overview.

The Open/Close Voice Commands Window command controls the display of the Voice Commands window for the current active character. If speech recognition services are disabled, this command is disabled. If speech recognition services are not installed, this command does not appear.

The Hide command plays the animation assigned to the character's Hiding state and then hides the character. If the character is already hidden, there will be a Show command instead.

The commands for the character's active client application follow the Hide command, preceded by a separator. Then the names of other applications using the character appear, also preceded by a separator.

The Notification Icon

If a character has been authored to include an icon, or an icon is generated for the character, the icon appears in the notification area of the Windows® taskbar when the character is loaded. This icon provides access to the character's pop-up menu, which provides access to Double Agent's pre-defined commands such as those that hide and show the character.

Moving the pointer over the notification icon displays a tip window that reflects the name of the character (in the character's current language), or the tip text specified by the character's active client application. Single-clicking the character's notification icon displays the character. The action associated with double-clicking the notification icon depends on the application currently controlling that character.

Right-clicking the notification icon displays the pop-up menu. When the character is visible, the pop-up menu displays the same commands as those displayed when right-clicking the character. If the character is hidden, only the Open Voice Commands Window or Close Voice Commands Window and Show commands appear.

The Word Balloon

In addition to spoken audio output, the Double Agent interface supports textual captioning in the form of text output in cartoon-style word balloons. Words appear in the word balloon as they are spoken. The word balloon hides when spoken output is completed. The Advanced Character Options window provides options to disable the word balloon's display as well as control attributes related to its appearance.

The Listening Tip

If speech input is enabled, a special tooltip window appears when the user presses the Listening key to begin voice input. The Listening tip displays contextual information related to the current input state of Double Agent. If compatible speech recognition has not been installed or has been disabled, the Listening tip does not appear.

The Voice Commands Window

If a compatible speech engine is installed, Double Agent supplies a special window called the Voice Commands Window that displays the commands that have been voice-enabled for speech recognition. The Voice Commands Window serves as a visual prompt for what can be spoken as input (commands cannot be selected with the mouse).

* Commands listed are based on the input-active client. For more information on defining commands that appear in the pop-up menu, see the Double Agent API Overview.

** Entries listed are all other applications currently hosting the character. For more information on defining this entry, see the Double Agent API Overview.

The window appears when a user selects the Open Voice Commands Window command, either by speaking the command or right-clicking the character and choosing the command from the character's pop-up menu. However, if the user disables speech input, the Voice Commands Window is not available.

The Voice Commands Window displays voice-enabled commands as an indented list. If the current hosting application supplies voice commands, they appear expanded and at the top of the window. Entries also appear for other applications using the character. The window also includes the Global Voice Commands predefined by Double Agent. If the current hosting application has no voice commands, the Global Voice Commands appear expanded and at the top of the window.

The user can size and move the Voice Commands Window. Double Agent remembers the last location of the window and redisplays it at that location if the user closes and re-opens the window. If the entries in the window exceed the current display size of the window, scroll bars appear.
