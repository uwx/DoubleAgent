Double Agent API

The Command Object

Description

A Command Object is an item in a Commands Collection. Double Agent provides the user access to your Command Objects when your client application becomes input-active.

Properties

Caption Specifies the text displayed for a Command in the character's pop-up menu.
ConfidenceText Specifies the text that appears in the Listening tip.
ConfidenceThreshold Specifies if the command's ConfidenceText appears in the Listening tip.
Enabled Specifies if the Command is enabled in the character's pop-up menu.
Name The unique name that identifies the Command Object in the Commands Collection.
Visible Specifies if the Command is visible in the character's pop-up menu.
VoiceCaption Specifies the text displayed for the Command Object in the Voice Commands window.
VoiceGrammar Specifies the grammar text that is passed to the speech recognition engine for matching this Command.

Remarks

A command may appear in either the character's pop-up menu and the Voice Commands window, or in both. To appear in the pop-up menu it must have a Caption and have the Visible property set to True. In addition, its Commands Collection's Visible property must also be set to True. To appear in the Voice Commands window, a Command must have its Caption and VoiceGrammar properties set.

When Double Agent receives input for one of your commands, it sends a Command event, and passes back the name of the Command as an attribute of the UserInput Object. You can then use conditional statements to match and process the Command.

Note that a character's pop-up menu entries do not change while the menu is displayed. If you add or remove commands or change their properties while the character's pop-up menu is displayed, the menu reflects those changes whenever the user next shows it. However, the Voice Commands window dynamically reflects any changes you make.

Referencing the Command Object

  • To access the properties of a Command Object, you reference it in its collection using its Name property. In VBScript and Visual Basic you can use the Name property directly: 

    control.Characters("CharacterID").Commands("Name>").property[ = value]

  • For programming languages that don't support collections, use the Command method: 

    control.Characters("CharacterID").Commands.Command("Name").property[ = value]

  • You can also use a Command Object by creating a reference to it. In Visual Basic, declare an object variable and use the Set statement to create the reference: 

    Dim Command1 as Object
...
Set Command1 = control.Characters("MyCharacterID").Commands("SampleCommand")
...
Command1.Enabled = True

  • You can also declare the object as type DaCtlCommand and create the reference. This convention enables early syntax checking and binding, which results in better performance: 

    Dim Command1 as DaControl.DaCtlCommand
...
Set Command1 = control.Characters("MyCharacterID").Commands("SampleCommand")
...
Command1.Enabled = True

  • In VBScript, you can't declare a reference as a particular type, but you can still declare the variable and set it to the Command in the collection: 

    Dim Command1
...
Set Command1 = control.Characters("MyCharacterID").Commands("SampleCommand")
...
Command1.Enabled = True
