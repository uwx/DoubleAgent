Double Agent API

The Activate Method

Description

Sets the active client or character.

Syntax

Visual Basic .NET

[Success = ]control.Characters ("CharacterID").Activate (State)

Visual Basic 6

[Success = ]control.Characters ("CharacterID").Activate[ State]

VBScript

[Success = ]control.Characters ("CharacterID").Activate[ State]

Visual C#

[Success = ]control.Characters ("CharacterID").Activate (State)

Visual C++

character.Activate (State, &Success)

Part Description
State You can specify one of the following values for this parameter:
0Not the active client
1The active client
2The input-active character (default)
Success True if the method succeeded.

Remarks

Avoid calling this method directly after a Show method. Show automatically sets the input-active client. When the character is shown, the Activate call may fail if it gets processed before the Show method completes.

Note:  Calling this method with State set to 1 does not typically generate an ActivateInput event unless there are no other characters loaded or your application is already input-active.


Details

When multiple characters are visible, only one of the characters receives speech input at a time. Similarly, when multiple client applications share the same character, only one of the clients receives mouse input (for example, Click or Drag events). The character set to receive mouse and speech input is the input-active character and the client that receives the input is the active client of that character. (The input-active character's window also appears at the top of the character window's z-order.) Typically, the user determines the input-active character by explicitly selecting the character. However, input activation also changes when a character is shown or hidden (the character becomes or is no longer input-active, respectively.)

You can also use this method to explicitly manage when your client receives input directed to the character such as when your application itself becomes active. For example, setting State to 2 makes the character input-active and your client receives all mouse and speech input events generated from user interaction with the character. Therefore, it also makes your client the input-active client of the character.

However, you can also set yourself to be the active client for a character without making the character input-active, by setting State to 1. This enables your client to receive input directed to that character when the character does become input-active. Similarly, you can set your client to not be the active client (not to receive input) when the character becomes input-active, by setting State to 0.

If you call this method as a function, it returns a boolean value that indicates whether the method succeeded. Attempting to call this method with the State parameter set to 2 when the specified character is hidden will fail. Similarly, if you set State to 0 and your application is the only client, this call fails because a character must always have a input-active client.


Example

Dim Genie as Object

Sub FormLoad()
    AgentControl.Characters.Load "Genie", "Genie.acs"

    Set Genie = AgentControl.Characters ("Genie")

    If (Genie.Activate = True) Then
      ' I'm active
    Else
      ' I must be hidden or something
    End If 
End Sub
