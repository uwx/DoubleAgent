Double Agent API

The LanguageID Property

Description

Specifies the character's language identifier.

Syntax

Visual C++

character->get_LanguageID (&LanguageID)

character->put_LanguageID (LanguageID)

Visual C#

[LanguageID = ]character.LanguageID

character.LanguageID = LanguageID

Visual Basic .NET

[LanguageID = ]character.LanguageID

character.LanguageID = LanguageID

Visual Basic 6

[LanguageID = ]character.LanguageID

character.LanguageID = LanguageID

VBScript

[LanguageID = ]character.LanguageID

character.LanguageID = LanguageID
Part Description
LanguageID A numeric value specifying the character's language ID. A character's language ID (LANGID) is a 16-bit value defined by Windows, consisting of a primary language ID and a secondary language ID.

Remarks

If you do not set the character's LanguageID, its language will be the current system language, if the corresponding Agent language DLL is installed. Otherwise, the character's language will be English (US).

This property also determines the language for word balloon text, the commands in the character'sPop-up Menu, and the speech recognition engine. It also determines the default language for text-to-speech output.

Details

If you try to set a character's LanguageID and the Agent language DLL for that language is not installed or a display font for the language is not available, Agent raises an error and LanguageID remains at its last setting.

Setting this property does not raise an error if there are no matching speech engines for the language. To determine if there is a compatible speech engine available for the LanguageID, check SRModeID or TTSModeID. If you do not set LanguageID, it will be set to the user default language.

Note: This property applies only to your client application's use of the character; the setting does not affect other clients of the character or other characters of your client application.

Note: If you set LanguageID to a language that supports bidirectional text (such as Arabic or Hebrew), but the system running your application does not have bidirectional support installed, text in the word balloon may appear in logical rather than display order.
