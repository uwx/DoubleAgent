Double Agent API

The CtlActiveClientChange Event

Description

Signalled when the active client of the character changes.

Syntax

Visual Basic .NET

Event control_CtlActiveClientChange (ByVal Sender As Object, ByVal E As CtlActiveClientChangeEvent)

Visual C#

delegate void control_CtlActiveClientChangeEventHandler (object Sender, CtlActiveClientChangeEvent E)
Part Description
Sender The source of the event.
E A CtlActiveClientChangeEvent Object that contains the event data:
CharacterID
The unique identifier of the character that has become active or inactive.
Active
True if the character has become the active client or False if it's no longer the active client.

Remarks

When multiple clients (AxControl instances) share the same character, the active client of the character receives mouse input (for example, click or drag events). Similarly, when multiple characters are displayed, the active client of the topmost character (also known as the input-active client) receives CtlCommand events.

A client application may receive this event when the user selects one of the client's entries in the character's pop-up menu or by voice command, when the client application changes its active status, or when another client terminates.

Double Agent sends this event only to the clients that are directly affected; that either become the active client or stop being the active client.
