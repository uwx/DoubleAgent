Double Agent API

The Listen Method

Description

Turns on Listening mode (speech recognition) for a timed period.

Syntax

Visual Basic .NET

[StartedListening = ]control.Characters ("CharacterID").Listen (Listen)

Visual Basic 6

[StartedListening = ]control.Characters ("CharacterID").Listen Listen

VBScript

[StartedListening = ]control.Characters ("CharacterID").Listen Listen

Visual C#

[StartedListening = ]control.Characters.this ["CharacterID"].Listen (Listen)

Visual C++

character->Listen (Listen, &StartedListening)
Part Description
Listen A boolean value that determines whether to turn Listening mode on or off.
StartedListening True if Listening mode was started.

Remarks

Setting the Listen parameter to True enables Listening mode (turns on speech recognition) for a fixed period of time (10 seconds). While you cannot set the value of the time-out, you can turn off Listening mode before the time-out expires.

If you (or another client) successfully set Listening mode on and you attempt to set this property to True before the time-out expires, the method succeeds and resets the time-out. However, if the Listening mode is on because the user is pressing the Listening key, the method succeeds, but the time-out is ignored and Listening mode ends based on the user's interaction with the Listening key.

This method succeeds only when called by the input-active client and only if speech services are available. To detect the success of this method, call it as a function and it will return a boolean value indicating whether the method succeeded. 

If Genie.Listen(True) Then
    ' The method succeeded
Else
    ' The method failed
End If

Details

This method fails if the user is pressing the Listening key and you attempt to set Listen to False. However, if the user has released the Listening key and Listening mode has not timed out, it will succeed.

Listen also fails if there is no compatible speech engine available that matches the character's LanguageID setting, the user has disabled speech input using the Advanced Character Options window, or the audio device is busy.

When you successfully set this method to True, the Double Agent signals the ListenStart event. Double Agent also signals the ListenComplete event when the Listening mode time-out completes or when you set Listen to False.

This method does not automatically call the Stop method and play a Listening state animation as Double Agent does when the Listening key is pressed. This allows you to determine whether to interrupt the current animation for the ListenStart event by calling Stop and playing your own appropriate animation. However, the Double Agent does call Stop and plays a Hearing state animation when a user utterance is detected.
