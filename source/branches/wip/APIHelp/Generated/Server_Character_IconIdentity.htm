Double Agent API

The IconIdentity Property

Description

Specifies the identity that Windows will use to remember the character's notification icon.

Syntax

Visual C++

character->get_IconIdentity (&IconIdentity)

character->put_IconIdentity (IconIdentity)

Visual C#

[IconIdentity =] character.IconIdentity

character.IconIdentity = IconIdentity

Visual Basic .NET

[IconIdentity = ]character.IconIdentity

character.IconIdentity = IconIdentity

Visual Basic 6

[IconIdentity = ]character.IconIdentity

character.IconIdentity = IconIdentity

VBScript

[IconIdentity = ]character.IconIdentity

character.IconIdentity = IconIdentity
Part Description
IconIdentity A string value that allows Windows to uniquely identify this character's notification icon. It must be either a valid GUID string or an empty string.

Remarks

Every notification icon has an identity that allows Windows to remember it's state (shown, hidden, active, etc.). By default, Windows associates a notification icon with the application that creates it.

You can supply either a GUID identifier, or an empty string to use the default identity.

  • A character's notification will be associated with the Double Agent server by default.
  • You can supply a single GUID for all characters you load. All of their notification icons will be associated with that single identity.
  • You can supply a unique GUID for each character you load. Windows will then identify each character's notification icon uniquely.

Note: This property applies only to your client application's use of the character; the setting does not affect other clients of the character or other characters of your client application.
