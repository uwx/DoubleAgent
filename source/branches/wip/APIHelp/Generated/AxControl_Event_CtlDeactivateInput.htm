Double Agent API

The CtlDeactivateInput Event

Description

Signalled when a client becomes non-input-active.

Syntax

Visual Basic .NET

Event control_CtlDeactivateInput (ByVal Sender As Object, ByVal E As CtlDeactivateInputEvent)

Visual C#

delegate void control_CtlDeactivateInputEventHandler (object Sender, CtlDeactivateInputEvent E)
Part Description
Sender The source of the event.
E A CtlDeactivateInputEvent Object that contains the event data:
CharacterID
The unique identifier of the character that has become non-input-active.

Remarks

A non-input-active client no longer receives mouse or speech events from Double Agent (unless it becomes input-active again). Double Agent sends this event only to the client that becomes non-input-active.

This event is signalled when your client application is input-active and the user chooses a another client's Command in a character's Pop-up Menu or the Voice Commands window, or you call the Activate method and set the State parameter to 0. It may also occur when the user selects the name of another character by clicking or speaking. You also get this event when your character is hidden or another character becomes visible.
