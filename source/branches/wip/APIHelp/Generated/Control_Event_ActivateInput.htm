Double Agent API

The ActivateInput Event

Description

Signalled when a client becomes input-active.

Syntax

Visual Basic .NET

Event control_ActivateInput (ByVal CharacterID As String)

Visual Basic 6

Sub control_ActivateInput (ByVal CharacterID As String)

VBScript

Sub control_ActivateInput (ByVal CharacterID)

Visual C#

delegate void control_ActivateInputEventHandler (string CharacterID)

Visual C++

void control_ActivateInput (LPCTSTR CharacterID)
Part Description
CharacterID The unique identifier of the character that has become input-active.

Remarks

The input-active client receives mouse and speech input events for the character. Double Agent sends this event only to the client that becomes input-active.

This event can occur when the user uses your Commands Collection, for example, by choosing a Commands Collection entry in the Voice Commands window or in the Pop-up Menu window for a character. It can also occur when the user selects a character (by clicking or speaking its name), when a character becomes visible, or when the character of another client application becomes hidden.

You can also call the Activate method (with State set to 2) to explicitly make the character topmost, which results in your client application becoming input-active and triggers this event. However, this event does not occur if you use the Activate method only to specify whether or not your client is the active client of the character.
