Double Agent API

The Control Object

The Double Agent control has its own methods, properties and events, and provides access to most other objects supported by Double Agent.

  • The Control object has the following methods:

    FindCharacterSREngines, FindCharacterTTSEngines, FindSREngines, FindTTSEngines, GetCharacterSREngine, GetCharacterTTSEngine, ShowDefaultCharacterProperties

  • The Control object has the following properties:

    AutoConnect, AutoSize, CharacterStyle, Connected, ControlCharacter, RaiseRequestErrors

    It also includes properties to retrieve other Double Agent objects:

    CharacterFiles, Characters, CommandsWindow, PropertySheet, Settings, SREngines, TTSEngines,

  • The Control object signals several events that enable your client application to track its state:

    ActivateInput, ActiveClientChange, AgentPropertyChange, BalloonHide, BalloonShow, Bookmark, Click, Command, DblClick, DeactivateInput, DefaultCharacterChange, DragComplete, DragStart, Hide, IdleComplete, IdleStart, ListenComplete, ListenStart, Move, RequestComplete, RequestStart, Show, Size

Remarks

Some programming environments may assign additional design-time or run-time properties. For example, Visual Basic adds a Name property that identifies the control, and also Left and Top properties that define the location of the control on a form, even if the control doesn't appear on the form's page at run time.

Note:  Double Agent does not support the Restart, Shutdown and HelpComplete events.
