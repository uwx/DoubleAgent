Double Agent API

The ActiveState Property

Description

Indicates if your application is the character's active client and if the character isinput-active.

Syntax

Visual C++

character->get_ActiveState (&ActiveState)

Visual C#

[ActiveState = ]character.ActiveState

Visual Basic .NET

[ActiveState = ]character.ActiveState

Visual Basic 6

[ActiveState = ]character.ActiveState

VBScript

[ActiveState = ]character.ActiveState
Part Description
ActiveState A member of the ActiveStateType Enumeration indicating the state of the character and your client application:
0 = Inactive Your client application is not the character's active client.
1 = Active Your client application is the character's active client.
2 = InputActive The character is the input-active character and will receive user input.

Remarks

When multiple client applications share the same character, the character's active client receives mouse input (for example, Double Agent control Click or Drag events). Similarly, when multiple characters are displayed, the active client of the topmost character (also known as the input-active client) receives Command events.

You can use the Activate method to set whether your application is the character's active client or to make your application the input-active client (which also makes the character topmost).
