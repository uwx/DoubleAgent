Double Agent API

Accessing the Control from Programming Languages

You can use Double Agent's control from Visual Basic 6, Visual Basic .NET, Visual C#, Visual C++, and other programming languages.

  • For programming languages that support native Windows programming, use Double Agent's native ActiveX control.
    • Make sure the language fully supports the ActiveX control interface, and follow its conventions for adding and accessing ActiveX controls.
    • The native control is registered as an OLE ActiveX Control object. If the programming language provides a list of registered OLE Automation objects, it will appear in the list.
    • You can also use the DoubleAgent.Control or DoubleAgent.Control.1.1 ProgID, or the {1147E530-A208-11DE-ABF2-002421116FB2} ClassID.
  • For programming languages that support the Windows Forms feature of the .NET Framework, there are two options:
    • You can use Double Agent's ActiveX Control Primary Interop Assembly (DoubleAgent.AxControl.dll).
    • It provides the Double Agent Control as a Windows Forms Control (a subclass of System.Windows.Forms.Control) that you can drop in any form.
    • Use the AxControl object in the DoubleAgent.AxControl namespace.
    • You can use Double Agent's Control Primary Interop Assembly (DoubleAgent.Control.dll).
    • It provides the Double Agent Control as simple a .NET object (a subclass of System.Object) that you can instantiate using the New operator (no form required).
    • Use the Control object in the DoubleAgent.Control namespace.

To access the control, Double Agent must already be installed on the target system. If you want to install Double Agent as part of your application, you can download Double Agent's Microsoft Installer installation (.MSI) file from the Web site. You can include this file in your installation setup program. Whenever it is executed, it will automatically install Double Agent on the target system.
