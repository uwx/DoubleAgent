Double Agent API

Using the Control in Applications

You can access Double Agent's control from Visual Basic 6, Visual Basic .NET, Visual C#, Visual C++, and other programming languages.

For programming languages that support native Windows programming, use Double Agent's native ActiveX® control.

  • Make sure the language fully supports the ActiveX® Control Interface, and follow its conventions for adding and accessing ActiveX® controls.
  • The native control is registered as an ActiveX® Control object. If the programming language provides a list of registered OLE Automation objects, it will appear in the list.
  • If the language supports it, you can also use the New operator with the "DoubleAgent.Control" or "DoubleAgent.Control.1.1" ProgID, or the "{1147E530-A208-11DE-ABF2-002421116FB2}" ClassID.

For programming languages that support the System.Windows.Forms namespace of the .NET Framework, there are two options:

  1. You can use Double Agent's ActiveX® Control Primary Interop Assembly (DoubleAgent.AxControl.dll).
    • It provides the Double Agent Control as a Windows Forms Control (a subclass of System.Windows.Forms.Control) that you can drop in any form.
    • Use the AxControl class in the DoubleAgent.AxControl namespace.
  2. You can use Double Agent's Control Primary Interop Assembly (DoubleAgent.Control.dll).
    • It provides the Double Agent Control as simple a .NET object (a subclass of System.Object) that you can instantiate using the New operator (no form required).
    • Use the Control class in the DoubleAgent.Control namespace.
