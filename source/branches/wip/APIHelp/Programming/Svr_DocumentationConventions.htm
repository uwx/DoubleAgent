Double Agent API

Documentation Conventions

This documentation uses the following typographical conventions:

Convention Description
return, Visible, Caption Words in bold are keywords.
server, String, Now Italic words are placeholders for information you supply.
commands A class name in italics represents a Double Agent Object or Collection instance.
Enter, F1 Words in all capital letters indicate filenames, key names, and key sequences.
commands->put_Enabled (VARIANT_TRUE) Words in this font are code samples.
\\ This is a comment An double backslash (\\) indicates a code comment.
[words or expression] Items inside square brackets are optional.
This | That A vertical bar indicates a choice between two or more items.

The programming examples in this documentation follow the conventions for C/C++. However, they are generally applicable to other languages as well.

This document assumes an understanding of the Component Object Model (COM) and/or the .NET Common Language Runtime (CLR).
