Double Agent API

Server Programming Overview

Because Double Agent is binary compatible with Microsoft® Agent, you can develop an Agent application using the Microsoft® Agent API. When the application is deployed:

  • On a system where only Microsoft® Agent is installed, the Microsoft® Agent run-time will be used.
  • On a system where Double Agent is also installed, but it is not emulating Microsoft® Agent, the Microsoft® Agent run-time will be used.
  • On a system where Double Agent is also installed, and it is emulating Microsoft® Agent, the Double Agent run-time will be used.
  • On a system where only Double Agent is installed, the Double Agent run-time will always be used.

You can also develop specifically for the Double Agent API. The remainder of this documentation is dedicated to that scenario.

To use the server interfaces, Double Agent must already be installed on the target system. If you want to install Double Agent as part of your application, you can download Double Agent's installation (.MSI) package from the Web site. You can include this file in your installation setup program (with the required acknowledgement). Whenever it is executed, it will install Double Agent on the target system.

To program the server, you will also need to install at least the Application Interfaces component of the Double Agent SDK.

Once Double Agent and its Application Interfaces components are installed, the following development files will be available in the installation folder:

File Description
Dev\DaServer.tlb The Double Agent Server COM type library 1
Dev\DaServerOdl.h The generated C/C++ header file for the Double Agent Server COM type library
Dev\DaServerInterface.h C++ Smart pointer definitions for the Double Agent Server type library (includes DaServerOdl.h)
Dev\DoubleAgent.Server.dll The Double Agent Server Primary Interop Assembly 2
Dev\DoubleAgent.Server.xml Microsoft® Visual Studio Intellisense data for DoubleAgent.Server.dll
Dev\AgentSvr.tlb The Microsoft® Agent Server COM type library 3
1 When Double Agent is installed, the server's COM components are added to the registry. This is a copy of the server type library. It's installed to be used as a reference in native Windows development projects, but is not required at run time.
2 When Double Agent is installed, this .NET Assembly is in installed in the Global Assembly Cache. This is a copy of the assembly. It's installed to be used as a reference in .NET development projects, but is not required at run time.
3 This file is only installed if Microsoft® Agent is not installed. It's purpose is to support programming to the Microsoft® Agent Server API on a system where Microsoft® Agent isn't present.
