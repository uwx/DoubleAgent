Double Agent API

Double Agent API Overview

The Double Agent API reproduces and extends the features of the Microsoft® Agent API.

It consists of two main components:

  • A Component Object Model Server that allows multiple applications, called clients or client applications, to access its animation, input, and output services at the same time. A client can be any application that connects to the Double Agent server's COM interfaces.

    As a COM server, the Double Agent server automatically starts only when a client application uses its COM interfaces and connects to it. It remains running until all clients close their connections. When no connected clients remain, the Double Agent server automatically shuts down.

  • A Microsoft ActiveX® Control that makes it easy to access Double Agent's services from programming languages that support the ActiveX® Control Interface.

    The control can be connected to the server, allowing Agent characters to be shared among multiple applications. It can also run standalone, providing Agent characters that are private to a specific application. This is a new feature of Double Agent that is not supported by Microsoft® Agent.

    The control also allows Double Agent to provide it's character animation services within a child window of a client application's window. This is a new feature of Double Agent that is not supported by Microsoft® Agent.

See the Double Agent Server and Control interfaces for more information on the specific API.

In addition to supporting stand-alone programs written for Windows®, Double Agent can be scripted to support Web pages, provided that the browser supports OLE Automation or the ActiveX® Interface. Microsoft Internet Explorer® and other browsers include support for ActiveX® as well as scripting languages that you can use to program Double Agent.
