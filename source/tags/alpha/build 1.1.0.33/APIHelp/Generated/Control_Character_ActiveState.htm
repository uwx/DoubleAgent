Double Agent API

The ActiveState Property

Description

Returns whether your application is the active client of the character and whether the character is input-active.

Syntax

Visual Basic .NET

[ActiveState = ]control.Characters ("CharacterID").ActiveState

Visual Basic 6

[ActiveState = ]control.Characters ("CharacterID").ActiveState

VBScript

[ActiveState = ]control.Characters ("CharacterID").ActiveState

Visual C#

[ActiveState = ]control.Characters.this ["CharacterID"].ActiveState

Visual C++

character.get_ActiveState (&ActiveState)

Part Description
ActiveState An value from the ActiveStateType enumeration indicating the state of the character and your client application:
0 = Inactive Your client application is not the character's active client.
1 = Active Your client application is the character's active client.
2 = InputActive The character is the input-active character and will receive user input.

Remarks

When multiple client applications share the same character, the active client of the character receives mouse input (for example, Double Agent control Click or Drag events). Similarly, when multiple characters are displayed, the active client of the topmost character (also known as the input-active client) receives Command events.

You can use the Activate method to set whether your application is the active client of the character or to make your application the input-active client (which also makes the character topmost).
