Double Agent API

The VisibilityCause Property

Description

Returns a VisibilityCauseType Enumeration value that specifies what caused the character's visible state to change.

Syntax

Visual Basic .NET

[VisibilityCause = ]control.Characters ("CharacterID").VisibilityCause

Visual Basic 6

[VisibilityCause = ]control.Characters ("CharacterID").VisibilityCause

VBScript

[VisibilityCause = ]control.Characters ("CharacterID").VisibilityCause

Visual C#

[VisibilityCause = ]control.Characters.this ["CharacterID"].VisibilityCause

Visual C++

character.get_VisibilityCause (&VisibilityCause)

Part Description
VisibilityCause

A VisibilityCauseType value:

0 = NeverShown The character has not been shown.
1 = UserHid The user hid the character using the command on the character's taskbar icon pop-up menu or using speech input.
2 = UserShowed The user showed the character using the command on the character's taskbar icon pop-up menu or using speech input.
3 = ProgramHid Your application hid the character.
4 = ProgramShowed Your application showed the character.
5 = OtherProgramHid Another client application hid the character.
6 = OtherProgramShowed Another client application showed the character.

Remarks

You can use this property to determine what caused the character to change visibility when more than one application is sharing (has loaded) the same character. These values are the same as those received by the Show and Hide events.
